Phase E — Configuration des Plugins (Post-Install)
Une fois WordPress en ligne, connecte-toi à l'admin (/wp-admin) :

1. Redis Object Cache
a- Installe le plugin Redis Object Cache.
b- Comme nous avons déjà injecté les constantes WP_REDIS_SENTINEL dans le wp-config.php, clique simplement sur "Enable Object Cache".
c- Vérifie dans l'onglet "Diagnostics" qu'il voit bien les 3 nœuds Sentinel.

2. MinIO (WP Offload Media Lite)
a- Installe le plugin WP Offload Media Lite.
b- Sélectionne Amazon S3 comme fournisseur (MinIO est compatible S3).
c- Dans "Connection Method", il devrait détecter tes clés définies dans AS3CF_SETTINGS.
d- Renseigne le bucket : wordpress-media.
e- Crucial : Change le "Custom Endpoint" pour : http://minio-ha:9000.

Pourquoi cette architecture est robuste ?
+ Galera : Si le pod mariadb-galera-0 meurt, WordPress bascule sur le service mariadb-galera qui pointe vers les survivants.
+ Sentinel : Si le Master Redis tombe, Sentinel élit un nouveau Master en < 10s. WordPress suit grâce au plugin.
+ MinIO : Ton stockage média est distribué sur 4 pods (Erasure Coding). Même avec 1 pod MinIO hors ligne, tes images restent servies.
+ Longhorn RWX : Si tu mets à jour un thème sur le Pod A, le Pod B et C voient la modification instantanément car ils partagent le même volume NFS interne géré par Longhorn.
